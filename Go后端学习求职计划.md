## 为什么 Go 对你是最优选？

- Go 只有 **25 个关键字**，Java 有 50+，语法复杂度不到 Java 一半
- Go 应届生简历数量远少于 Java，面试没那么卷
- 用 Go 的公司大多是云原生/中间件/SaaS 方向，技术氛围好
- 二线城市 Go 岗位在增长（武汉/成都/南京尤其明显）

## 二、第一个月（6月中 - 7月中）：Go 基础打牢

### Go 语言基础（2周，每天 4-6 小时）

跟 Java 不同，Go 基础部分很短很干净，两周够用了。

**第一周：基础语法**

- ★ 变量声明（var、:=）、基本类型（int、string、bool、float）
- ★ 数组 vs 切片（slice）——重点理解底层数组扩容
- ★ map（字典）的使用
- ★ 函数：多返回值、命名返回值、defer
- ★ 指针：Go 只有指针没有引用，和 C 类似但更简单
- ★ struct（结构体）和方法
- ★ interface（接口）：Go 的接口是隐式实现的，和 Java 完全不一样
- ★ 错误处理：Go 没有 try-catch，用 error 返回值

**第二周：并发 + 标准库**

- ★ goroutine + channel —— Go 的灵魂，必须彻底搞懂
- select 多路复用
- sync 包：WaitGroup、Mutex、sync.Map
- ★ 标准库常用包：fmt、net/http、encoding/json、time、strconv、os、io
- 用 net/http 手写一个简单的 HTTP 服务（不用框架）

**怎么学**：
- B站搜「老男孩 Go 语言基础」或者「bilibili Go 语言零基础」（哪个看着顺眼看哪个）
- **必看**：Go 官方文档 [A Tour of Go](https://go.dev/tour/)，在线交互式，半天过完一遍
- 每天把学到的知识点自己敲一遍，不要复制粘贴
- 重点练 goroutine + channel 的小 demo

### MySQL（1周，每天 3-4 小时）

和 Java 路线一样，这部分不变：

**重点**：
- ★ CRUD 语句、多表联查（JOIN/LEFT JOIN/子查询）
- ★ 索引原理（B+树），索引失效的场景
- ★ 事务 ACID，隔离级别
- SQL 优化基础（explain）

**怎么练**：
- 装 MySQL，命令行敲 SQL
- B站「尚硅谷 MySQL」基础部分
- LeetCode 刷 SQL 题（免费 50 道经典题，全做一遍）

### 数据结构与算法（穿插，每天 1 小时，长期坚持）

二线城市 Go 岗位算法面试不难，但基本的数据结构必须会：

**必会**：
- 数组、切片、链表、map
- 栈、队列
- 二叉树遍历（递归写法）
- 排序（快排思想，能讲就行）
- 哈希表

**刷题策略**：
- LeetCode 按标签刷 easy，每天 2 道
- 优先 Top 100 里的题
- 用 Go 写，一开始不习惯可以先用 Python 理解思路再翻 Go

## 三、第二个月（7月中 - 8月中）：框架 + 项目

### Go Web 框架 —— Gin（1.5周）

Go 生态里 Gin 就是标准答案，90% 的 Go 后端都用它。

**重点**：
- ★ 路由分组、RESTful API 设计
- ★ 中间件（Middleware）：日志、跨域、鉴权
- ★ 参数绑定与校验（ShouldBind、binding tags）
- ★ 统一响应格式封装
- ★ 全局错误处理/panic 恢复
- 表单/JSON 数据接收
- 文件上传

**怎么学**：
- B站搜「Gin 框架入门」或「李文周 Gin 教程」
- 跟着视频把每个 demo 敲一遍
- [Gin 官方文档](https://gin-gonic.com/zh-cn/docs/) 很清晰，边看边写

### ORM —— GORM（0.5周）

Go 的 ORM 没有 Java 的 MyBatis-Plus 那么复杂，GORM 基本就够用。

**重点**：
- ★ 模型定义、自动建表
- ★ CRUD 操作
- ★ 关联查询（Has One / Has Many / Belongs To）
- 分页查询
- 事务操作
- 钩子函数（了解）

### Redis（1周）

**重点**：
- ★ 5 种基本数据类型及应用场景
- ★ 缓存穿透/击穿/雪崩 是什么 + 解决方案
- ★ Go 操作 Redis：go-redis 库
- ★ Gin + Redis 缓存中间件实现
- 过期策略（了解）

### 项目实战（开始做）

**项目：校园求职/招聘平台后端（Go 版）**

用 Go + Gin + GORM 实现，比 Java 的 Spring Boot 轻量很多，代码量也少。

**功能模块**（同上，但用 Go 写）：
1. **用户模块**：注册、登录（JWT）、角色区分（学生/企业）
2. **职位模块**：CRUD、分页+条件搜索
3. **简历模块**：上传、查看、修改（文件上传到本地/OSS）
4. **投递模块**：投递+查看状态
5. **收藏模块**：收藏/取消收藏
6. **基础建设**：统一响应封装、全局异常中间件、Swagger 文档（gin-swagger）

**技术栈**：Go 1.21+ + Gin + GORM + MySQL + Redis + JWT + Swagger

**进度安排**：
- Week 1：项目骨架搭建（分层：router → controller → service → dao）+ 用户模块 + JWT
- Week 2：职位模块 + 简历模块
- Week 3：投递模块 + 收藏模块 + Redis 缓存
- Week 4：Swagger 文档、完善细节、README、推到 GitHub

**项目结构参考**：
```
campus-recruit/
├── config/          # 配置文件
├── controller/      # 控制器（处理请求）
├── service/         # 业务逻辑
├── dao/             # 数据库操作
├── model/           # 数据模型
├── middleware/      # 中间件（JWT、日志、跨域等）
├── router/          # 路由
├── utils/           # 工具函数
├── main.go          # 入口
└── README.md        # 项目说明
```

### 面试题集中刷（8月中-9月）

Go 面试比 Java 短很多，核心就这些：

**Go 语言本身**（最重要，占面试 50%）：
- goroutine 和线程的区别
- channel 的原理、有缓冲 vs 无缓冲
- GMP 调度模型（务必搞懂，高频考点）
- GC 机制（三色标记，大概讲清楚就行）
- slice 扩容机制
- map 并发安全吗？怎么解决？
- defer 执行顺序
- interface 内部实现（iface/eface）
- Go 的逃逸分析（了解）

**数据库**（20 道）：
- 索引原理、最左匹配
- SQL 优化思路
- 事务隔离级别
- MySQL 锁（行锁/表锁）

**Redis**（15 道）：
- 缓存三兄弟（穿透/击穿/雪崩）
- 5 种数据类型场景
- 过期策略

**计算机网络 + 操作系统**（10 道，二线问得少）：
- TCP 三次握手四次挥手
- HTTP 1.0/1.1/2.0 区别
- 进程 vs 线程 vs 协程（goroutine）

**算法**（大概率不考太难）：
- 常见排序讲思想
- easy 级算法题手写

## 五、秋招投递策略（9月-11月）

### Go 岗位在哪里？

Go 岗位不像 Java 那么散，集中在特定类型的公司：

| 公司类型 | 代表企业（二线城市有岗位的） |
|----------|---------------------------|
| 云原生/基础设施 | 七牛云、又拍云、青云 |
| 电商/SaaS | 微盟、有赞、光云科技 |
| 游戏后端 | 米哈游（上海）、莉莉丝、各种中小游戏公司 |
| 互联网中厂 | 字节跳动（成都/武汉）、B站（上海）、小红书（武汉） |
| 银行/金融科技 | 新网银行（成都）、各类金融科技公司 |
| 外包/乙方 | 中软、软通等（也有 Go 项目） |

### 二线城市 Go 岗位分布

| 城市 | 情况 |
|------|------|
| 成都 | Go 岗位最多的二线城市，字节成都有 Go 岗 |
| 武汉 | 斗鱼、小米武汉、各类 SaaS |
| 南京 | 润和、各种云计算公司 |
| 西安 | ThoughtWorks 西安、各类外包 |
| 长沙 | 芒果TV、万兴科技 |
| 杭州 | (新一线) Go 岗位巨多，但房价也高 |

### 投递渠道

1. **BOSS 直聘**搜「Go 校招」「Golang 应届」「Go 后端」
2. **牛客网**搜 Go 面经 + 投递
3. **拉勾/猎聘**搜 Golang
4. 公司官网直投
5. **校园招聘会**别错过

### 投多少？

- 目标：**60-100 家**
- Go 不比 Java 岗位多，但投递的人也少，竞争比 Java 小
- 同理外包也要投，先入行

### 心态

- Go 应届生简历池小，你做完一个完整项目已经赢了大多数
- 二线城市 Go 岗位虽然总量不如 Java，但每年投的人少，**面试机会反而更多**
- 秋招不理想还有春招（2-4月），时间完全够
- 第一份工作入行即胜利
